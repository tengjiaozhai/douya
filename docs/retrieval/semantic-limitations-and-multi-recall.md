# 语义相似度检索的局限与为什么需要多路召回

## 先用一句话理解

语义相似度检索很擅长找“意思接近”的内容，但它不总能保证“词、数字、时间、规则”都对。

## 语义相似度检索现在的局限

1. 它更懂大意，不一定懂细节。  
   例子：你要“2024 年之后发布的版本”，它可能召回“版本介绍很像，但发布时间不满足”的内容。

2. 对精确词不稳定。  
   例子：型号 `A17 Pro`、药名、人名、股票代码这类词，向量检索可能匹配到“类似但不相同”的结果。

3. 对否定和约束容易出错。  
   例子：“不要缓存”“不包含广告”这类条件，有时会被当成普通语义，导致误召回。

4. 会被压缩损失信息。  
   一段长文本被压成一个向量，局部关键信息可能被冲淡。

5. 为了速度会做近似搜索，可能漏掉正确结果。  
   ANN 检索很快，但不是每次都能找到全量最优邻居。

6. 分数可解释性弱，业务调参不直观。  
   你知道“像”，但不容易解释“为什么像到这个分数”。

## 为什么需要多路召回

核心原因：**不同召回路有不同强项，组合后能互补。**

1. 关键词召回补“精确命中”。  
   适合型号、数字、缩写、专有名词。

2. 语义召回补“同义表达”。  
   适合同义词、口语化改写、自然语言问题。

3. 规则/过滤召回补“硬约束”。  
   适合时间范围、状态、权限、地域、白名单等条件。

4. 热门或个性化召回补“业务目标”。  
   适合点击率、转化率、用户兴趣等目标。

## 推荐的简单流程

1. 多路并行召回候选（keyword + semantic + rule + personalized）。
2. 统一去重与融合（可用加权或 RRF）。
3. 交给重排模型做最终排序。

## 结论

单路语义召回会“有盲区”。  
多路召回不是重复劳动，而是用互补能力减少漏召回、降低误召回、提升整体稳定性。
