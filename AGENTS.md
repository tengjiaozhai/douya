# Repository Rules

## Rule: README Sync Is Mandatory

1. 每次完成任何代码变更（包括 `src/`、`apps/`、`libs/`、`docs/`、配置文件、脚本）后，必须在仓库根目录 `README.md` 同步更新本次实现结果。
2. `README.md` 更新至少包含：
   - 变更摘要（做了什么）
   - 行为变化（对调用方/运行方式的影响）
   - 必要的配置变化（新增/删除配置项）
3. 如果本次改动没有新增功能，也必须在 `README.md` 的更新日志中补充最小记录，说明“重构/清理/修复”内容。
4. 该规则对所有会话生效（包括新会话）。

## Rule: Design Docs And Test Cases Must Be Archived Under docs/

1. 所有设计文档（架构设计、方案设计、技术选型、改造归档）必须存放在 `docs/` 目录下，禁止散落到其他业务目录。
2. 所有测试用例文档（测试方案、测试步骤、验收用例、联调用例）必须存放在 `docs/` 目录下，建议按主题归档到 `docs/<topic>/`。
3. 新增或迁移设计文档/测试用例时，文件命名应语义化并可追溯（建议包含模块名与用途）。
4. 若历史文档不在 `docs/` 下，后续改动时应优先补做迁移并在 `README.md` 更新日志记录。
5. 该规则对所有会话生效（包括新会话）。
